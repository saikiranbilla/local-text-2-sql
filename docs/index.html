Architecture

A compound AI pipeline.

Each agent is small, specialized, and verifiable. No black-box monoliths.

01

Semantic Resolver

Hybrid fuzzy-matching engine that prunes the full database schema down to only the tables relevant to the user's question. Prevents the LLM from hallucinating column names by reducing its search space before generation.

02

SQL Generator

Claude Haiku (or local Qwen2.5) generates a DuckDB-dialect SQL query from the pruned schema. Uses Anthropic Prompt Caching to reduce latency by up to 90% on repeated schema lookups. Supports coreference resolution for follow-up questions.

03

Critic Agent

A self-correcting execution loop. If the generated SQL throws an error, the Critic Agent receives the error message alongside the failed query and attempts a fix — up to 3 times. Guarantees deterministic outputs, not probabilistic ones.

04

Local Execution (DuckDB)

All data processing runs in-process via DuckDB — an embedded analytical database. No cloud warehouse. No data transmitted to third parties. Your CSV files stay on your machine. Results stream back to the UI in real-time via Server-Sent Events.
